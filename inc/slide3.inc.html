API

Application Programming Interface
A layer between two interacting software components.

REST

REpresentational State Transfer
An architectural style of programming; opinionated with a focus on statelessness and a separation of concerns.

Stateless?

  • No memory of a client between requests
    • clients are responsible for saving state
  • Requires a token to authenticate each request
  • Flexible, scalable, replicable

The Problem

  • Monolithic software design is difficult with teams
  • Developers may have different specializations
  • Higher risk of bugs and errors

Separation of Concerns

Using RESTful APIs, developers are able to maintain and update front-end and back-end codebases separately.

APIs can act as a single source of truth to multiple clients that have no relationship to eachother besides being consumers of the same API.
